# Security policy

## Supported versions

No production version has been released yet. A supported-version table will be
added with the first stable release.

## Reporting a vulnerability

Do not include vulnerability details, credentials, private addresses, or logs
containing personal information in a public issue.

Before the first public release, this document will provide a private reporting
channel and the expected acknowledgement process. Until then, do not deploy the
project on production infrastructure.

## Release verification

Official releases include a signed manifest and checksums. Installation
documentation will contain the release public key and independently published
fingerprint before downloads are offered.
